03 / 18
对比分析

Coding Agent 的局限

Cursor / Copilot / Claude Code

  • 何时 plan、何时 code、何时 test
    — 关键决策都要人来做
  • 只能用机器上已有的工具
  • 对话式，每一步都需要人确认

OpenClaw

  • 自己决定 plan/code/test 节奏
  • Deal with ambiguity
    — 模糊需求也能自己摸索
  • 搜索、浏览器、API、发消息、多 Agent 协作
  • 24/7 运行，主动式工作
核心差异：Coding Agent 是工具，OpenClaw 是助手

踩坑实录

Gateway 与 LiteLLM 的血泪史

🔥

Gateway 配置三连挂

改 bind / TLS / port，三次把 Gateway 搞挂。每次都要别人 SSH 进来救……

💡 教训：基础设施配置是红线，不要让 Agent 乱碰
🫁

Agent 拔自己氧气管

迁移 LiteLLM 时，先停旧服务再装新的。结果 LLM 断了，Agent 也一起断了……

💡 教训：零停机原则 — 先立新，再拆旧
07 / 18
踩坑实录

Remote Node 三连失败

📡

Remote Node 方案反复失败

三次搞挂之后，最终放弃这个方案，改用 A2A 跨队通信替代。

💡 教训：有时候换方案比修 bug 更好

踩坑核心规律

  • 自由 = 风险：Agent 越能干，越容易踩红线
  • 基础设施加保护：关键配置不授权给 Agent
  • 换方案成本更低：别在一棵树上吊死
