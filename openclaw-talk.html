03 / 16
对比分析

Coding Agent vs OpenClaw

    🔧 Cursor / Claude Code

    • 何时 plan / code / test — 人来决定
    • 只用机器上已有的工具
    • 对话式，每步需要人确认
    • 工作完就停了

    🦞 OpenClaw

    • 自己决定 plan/code/test 节奏
    • Deal with ambiguity — 模糊需求自己摸
    • 搜索、浏览器、API、消息、多 Agent
    • 24/7 主动运行，有 Heartbeat / Cron
核心差异：Coding Agent 是工具，OpenClaw 是助手 — 能自主处理模糊需求并直接交付

Setup 经验 2/2

Gateway 配置：碰都不要碰

🔥

踩坑：Gateway 三连挂

改了 bind / TLS / port 配置，三次把 Gateway 搞挂。每次都要别人 SSH 进来救……

💡 教训：Gateway 配置是红线，不要让 Agent 乱改。改前备份，确认可回滚
设置检查清单
  • 先用默认配置跑通再改
  • Gateway bind/port/TLS 修改前备份
  • 云部署开好安全组，确认端口
  • 别急，一步一步来
⚠️ Gateway 挂了 = 龙虾没氧气 = 什么都做不了，后果很严重

总结

Key Takeaways

  • 01 OpenClaw 是助手而不只是工具 — 能处理模糊需求，直接交付产品
  • 02 Gateway 是红线：乱改配置会断氧气。Setup 要慢、要稳、要能回滚
  • 03 稳定性靠三件套：多龙虾互救 + systemd Restart=always + 模型 fallback
  • 04 M2 管理模式让 Agent 真正成为团队 — context 干净，决策更好，效率更高
  • 05 知识共享（memory → memex → wiki）让整个团队持续进化
